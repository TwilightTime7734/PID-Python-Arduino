Auto Tune Outline
=================

Purpose
-------
- The app runs an adaptive roll/pitch excitation session, records the drone response in Blackbox, then analyzes the log for PID tuning recommendations.
- It does not appear to automatically write the recommended PID values during the auto session; it shows recommendations and suggested CLI commands for review.

Preflight
---------
- Requires the Arduino output link to be connected.
- Requires the flight controller link to be connected and producing attitude telemetry.
- Refuses to start while auto-level, another hold/pulse command, or Blackbox import/analyze is active.
- Shows a confirmation checklist for stand security, trainer mode, and clear area.

Adaptive Excitation
-------------------
- Creates an AdaptiveExcitationController and starts a timed control loop.
- The FC attitude poll runs continuously and records live roll/pitch samples.
- The controller chooses the lower-confidence axis: roll or pitch.
- It chooses the lower-confidence direction unless the drone is already tilted, then it biases commands back toward center.
- It sends short PWM offset pulses through the Arduino:
  - Force range: 100-260 us.
  - Hold range: 0.25-0.45 s.
  - Settle window: up to 0.55 s.
- Each pulse is tracked as an excitation event:
  - Axis and direction.
  - Command force and hold time.
  - Baseline angle.
  - Peak attitude change.
  - Response delay.
  - Final settle error.

Coverage Goal
-------------
- Tracks roll negative, roll positive, pitch negative, and pitch positive separately.
- A valid event has a peak response between 8 and 25 degrees.
- Each direction targets at least 6 valid events.
- Each direction also targets at least an 80% settle-success ratio.
- The session runs for at least 45 s and stops when all coverage targets are met.
- It stops no later than 180 s, even if confidence targets are not met.

Safety Behavior
---------------
- Aborts if FC telemetry is stale for more than 1 s.
- Reduces command force near the 35 degree soft limit.
- Enters recovery mode around 43 degrees and commands back toward center.
- Leaves recovery once roll and pitch are below 20 degrees.
- Aborts at the 45 degree hard limit.
- Restores base outputs after ending runtime.

Blackbox Pipeline
-----------------
- After session completion, the app disconnects the FC serial link and enters MSC mode.
- It imports or locates Blackbox logs under blackbox_imports.
- It analyzes the newest imported log when available; otherwise it analyzes the pulled log directory.
- INAV-Toolkit analysis checks PID response, gyro noise, motor behavior, D-term noise, phase lag, and related action items.
- If toolkit analysis is unavailable, the CSV fallback creates conservative PID hints from stick response traces.

Report Output
-------------
- The app builds a summary from the adaptive session payload and Blackbox analysis.
- The report includes confidence metrics, analysis summary, PID highlights, suggested CLI settings, warnings, and chart/viewer artifacts.
- Reports are written under blackbox_imports/reports/session_<timestamp>.

Main Code Areas
---------------
- modbus_app/app.py: UI orchestration, auto session runtime, Blackbox pipeline.
- modbus_app/adaptive_session.py: adaptive controller, coverage scoring, safety decisions.
- modbus_app/blackbox_import.py: Blackbox import, decode, PID recommendation extraction.
- modbus_app/auto_tune_report.py: summary and artifact generation.
- auto_session_flow.mmd: detailed flow diagram.
